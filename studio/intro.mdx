Rasa Studio is a no-code tool that lets you create, refine, and test Rasa assistants visually. Anyone on your team can build assistants, review conversational content, and run tests—all in one place—making collaboration straightforward regardless of technical background.

## Who is Studio for?

Studio works for anyone who prefers a visual approach to building an AI assistant. Think of it as an advanced content management system for conversational AI, one that lets you collaborate on the same assistant as your development team without getting in each other's way.

## How Studio Relates to Rasa Pro

Studio runs on top of Rasa Pro—the enterprise Rasa framework. Every assistant you build in Studio is backed by the same CALM (Conversational AI with Language Models) engine that powers production Rasa deployments.

At any point you can export your assistant's YAML files using the Rasa CLI and continue working locally:

```bash
rasa studio download <assistant-name>
```

This gives you full access to the underlying `config.yml`, `domain.yml`, and flow YAML files, so you can apply advanced configurations—such as custom NLU pipelines, IVR channels, end-to-end tests, or secret management—that are not yet available through the Studio UI.

<Note>
  Studio is a self-hosted product. You deploy it in your own environment using Docker Compose or Helm. See the [Installation guide](./installation) to get started.
</Note>
