Rasa Pro reads configuration from environment variables for sensitive values (API keys, passwords), runtime tuning (log levels, pool sizes), and feature flags (beta capabilities). This page lists every supported variable organized by category, with its description, default value, and whether it is required.

## How to Set Environment Variables

<Tabs>
  <Tab title="Shell export">
    ```bash
    export RASA_LICENSE=<your-license-key>
    export OPENAI_API_KEY=<your-openai-key>
    ```
  </Tab>
  <Tab title=".env file">
    Create a `.env` file in your project root (add it to `.gitignore`):

    ```bash
    RASA_LICENSE=<your-license-key>
    OPENAI_API_KEY=<your-openai-key>
    LOG_LEVEL=INFO
    ```

    Then load it before running Rasa:

    ```bash
    set -a && source .env && set +a
    rasa run
    ```
  </Tab>
  <Tab title="Docker">
    ```bash
    docker run \
      -e RASA_LICENSE=<key> \
      -e OPENAI_API_KEY=<key> \
      rasa/rasa:latest run
    ```
  </Tab>
  <Tab title="Kubernetes">
    Store secrets in a Kubernetes `Secret` object and mount them as environment variables in your pod spec:

    ```yaml
    env:
      - name: RASA_LICENSE
        valueFrom:
          secretKeyRef:
            name: rasa-secrets
            key: license
      - name: OPENAI_API_KEY
        valueFrom:
          secretKeyRef:
            name: rasa-secrets
            key: openai-api-key
    ```
  </Tab>
</Tabs>

## License

| Variable | Description | Default | Required |
|---|---|---|---|
| `RASA_LICENSE` | Your Rasa Pro license key. Must be set for Rasa Pro to start. | — | ✅ Required |

## LLM API Keys

These are the standard environment variables each LLM provider reads when no explicit `api_key` is set in `endpoints.yml`. You can also pass them directly in your model group config using `${VAR_NAME}` syntax.

| Variable | Provider | Required |
|---|---|---|
| `OPENAI_API_KEY` | OpenAI | Required for OpenAI |
| `AZURE_API_KEY` | Azure OpenAI | Required for Azure |
| `MISTRAL_API_KEY` | Mistral | Required for Mistral |
| `GEMINI_API_KEY` | Google Gemini (AI Studio) | Required for Gemini |
| `ANTHROPIC_API_KEY` | Anthropic (Claude) | Required for Anthropic |
| `HUGGINGFACE_API_KEY` | HuggingFace Inference Endpoints | Required for HuggingFace |
| `GROQ_API_KEY` | Groq | Required for Groq |
| `COHERE_API_KEY` | Cohere | Required for Cohere |
| `TOGETHERAI_API_KEY` | Together AI | Required for Together AI |
| `VOYAGE_API_KEY` | Voyage AI (embeddings) | Required for Voyage AI |
| `AWS_ACCESS_KEY_ID` | Amazon Bedrock | Required for Bedrock (if not using IAM role) |
| `AWS_SECRET_ACCESS_KEY` | Amazon Bedrock | Required for Bedrock (if not using IAM role) |
| `AWS_REGION` | Amazon Bedrock | Required for Bedrock |
| `AWS_SESSION_TOKEN` | Amazon Bedrock (temporary credentials) | Optional |

## SSL and Proxy

| Variable | Description | Default | Required |
|---|---|---|---|
| `RASA_CA_BUNDLE` | Path to a custom CA certificate bundle (PEM format) for TLS-intercepting proxies. Replaces the deprecated `REQUESTS_CA_BUNDLE`. | OS certificate store | Optional |

These variables control verbose prompt/response logging for specific CALM components:

| Variable | Scope | Default |
|---|---|---|
| `LOG_LEVEL_LLM` | All LLM-based components | `DEBUG` |
| `LOG_LEVEL_LLM_COMMAND_GENERATOR` | All `LLMCommandGenerator` variants | `DEBUG` |
| `LOG_LEVEL_LLM_ENTERPRISE_SEARCH` | `EnterpriseSearchPolicy` prompts | `DEBUG` |
| `LOG_LEVEL_LLM_INTENTLESS_POLICY` | `IntentlessPolicy` prompts | `DEBUG` |
| `LOG_LEVEL_LLM_REPHRASER` | `ContextualResponseRephraser` prompts | `DEBUG` |
| `LOG_LEVEL_NLU_COMMAND_ADAPTER` | NLU command adapter | `INFO` |
| `LOG_LEVEL_LLM_BASED_ROUTER` | `LLMBasedRouter` component | `INFO` |

## Beta Feature Flags

These variables enable features that are not yet generally available. Always check the documentation for any stability caveats before enabling them in production.

| Variable | Feature | Default |
|---|---|---|
| `RASA_PRO_BETA_FINE_TUNING_RECIPE` | Enable the LLM fine-tuning recipe (`rasa llm finetune prepare-data`). | `false` |
| `RASA_PRO_BETA_STUB_CUSTOM_ACTION` | Stub custom actions in E2E tests without running the action server. | `false` |
| `RASA_PRO_BETA_E2E_CONVERSION` | Convert sample conversations into E2E test cases automatically. | `false` |
| `RASA_PRO_BETA_REPEAT_COMMAND` | Enable the voice repair pattern that repeats the last bot message when asked. | `false` |
| `RASA_PRO_BETA_VIER` | Enable the `vier_cvg` voice connector. | `false` |
| `CLARIFY_ON_MULTIPLE_START_FLOWS` | Trigger clarification when multiple `StartFlow` commands are generated simultaneously. | `false` |

| Variable | Description | Default | Required |

| `RASA_ANALYTICS_DB_URL` | URL of the analytics data store. | — | Optional |
| `LOGGING_LEVEL` | Log level (`DEBUG`, `INFO`, `WARNING`, `ERROR`, `CRITICAL`). | `INFO` | Optional |
| `FORCE_JSON_LOGGING` | Use structured JSON logging. | — | Optional |

| `RASA_ANALYTICS_CONSUMER_COUNT` | Number of Kafka consumers. | `1` | Optional |
| `RUN_ANALYTICS_DB_MIGRATIONS` | Run analytics DB migrations on startup. | `true` | Optional |
